# 📌 Padronização do Projeto

Abaixo estão as diretrizes para padronização de nomes de **commits** e **branches** que devem ser seguidas pela equipe.

---

## 💬 Commits

Utilize prefixos semânticos para deixar claro o propósito da sua alteração. 

**Exemplos:**
* `feat: criação da funcionalidade xyz...`
* `fix: correção da funcionalidade xyz...`

> 💡 **Atenção:** Commit não é desabafo! Seja sucinto, objetivo e claro no que você escreve.

---

## 🌿 Branches

Para mantermos o repositório organizado, dividiremos as branches por trimestres.

**Exemplos:**
* `1_tri/insert-enemy`
* `2_tri/insert-characters`
* `3_tri/insert-player`

> 📁 **Regra de Organização:**
> * A cada trimestre, será utilizada uma "pasta" (ex: `1_tri/`) para agrupar as branches criadas no respectivo período.
> * O texto escrito à direita da barra (`/`) deve ser **direto e claro** sobre qual feature ou correção será trabalhada naquela branch.
